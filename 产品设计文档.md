# 英语学习App产品设计文档

## 一、项目概述

### 1.1 产品名称
**暂定名**：EasyEnglish 英语学习助手

### 1.2 产品定位
一款面向英语学习者的一站式免费学习应用，结合AI技术提供个性化、互动式的学习体验。

### 1.3 目标用户
- 英语初学者至中高级学习者
- 需要提升英语口语、写作、翻译能力的学生和职场人士
- 希望通过社区互动提高学习动力的用户

### 1.4 核心价值
- **免费使用**：降低英语学习门槛
- **AI驱动**：个性化学习路径和实时反馈
- **多功能集成**：一站式满足听、说、读、写需求
- **社区互动**：通过论坛交流提升学习动力

## 二、功能模块设计

### 2.1 单词学习模块

#### 功能描述
提供科学的单词记忆方法，帮助用户高效积累词汇量。

#### 核心功能
1. **单词库管理**
   - 内置词库：四六级、考研、雅思、托福、商务英语等
   - 自定义词库：用户可创建个人词库
   - 词库分类：按主题、难度、使用频率分类

2. **记忆算法**
   - 采用艾宾浩斯遗忘曲线算法
   - 智能复习提醒
   - 根据记忆状态动态调整复习频率

3. **学习模式**
   - 卡片记忆：正面单词、背面释义
   - 拼写练习：听音拼写、看释义拼写
   - 选择题测试：选出正确释义/单词
   - 例句学习：通过例句理解单词用法

4. **学习统计**
   - 每日学习量统计
   - 掌握程度分析
   - 学习曲线可视化

#### 技术实现
- 后端：Spring Boot + MySQL（单词数据存储）
- 算法：基于Sm2+算法优化记忆曲线
- 前端：React Native/Flutter（跨平台）

---

### 2.2 AI英语日常聊天模块

#### 功能描述
通过AI对话机器人，提供真实的英语对话练习环境。

#### 核心功能
1. **AI对话伙伴**
   - 支持多种对话场景：日常闲聊、商务对话、旅行英语等
   - AI角色设定：可切换不同口音、性格的AI伙伴
   - 实时语音对话：支持语音输入和AI语音回复

2. **语法和发音纠正**
   - 实时检测语法错误并提示
   - 发音评分和纠正建议
   - 用词不当提醒

3. **对话历史与复习**
   - 保存对话记录
   - 标记重要表达和生词
   - 支持回放和复习

4. **学习报告**
   - 词汇使用频率分析
   - 语法弱点识别
   - 进步趋势可视化

#### 技术实现
- AI模型：接入GPT-4/Claude等大模型API
- 语音识别：Google Speech-to-Text / Azure Speech Services
- 语音合成：Azure TTS / Amazon Polly
- 自然语言处理：Stanford CoreNLP / SpaCy

---

### 2.3 英语写作模块

#### 功能描述
提供英语写作练习和智能批改服务。

#### 核心功能
1. **写作练习**
   - 提供各类写作题目：议论文、书信、摘要等
   - 支持自由写作和命题写作
   - 字数统计和写作时间记录

2. **智能批改**
   - 语法错误检测与修正建议
   - 拼写检查
   - 用词丰富度分析
   - 句式多样性评估
   - 逻辑思维连贯性分析

3. **范文参考**
   - 提供各类题材的优秀范文
   - 范文结构分析
   - 好词好句摘录

4. **写作社区**
   - 用户可分享自己的作文
   - 社区互评功能
   - 老师/达人点评

#### 技术实现
- 语法检查：LanguageTool / Grammary API
- 文本分析：NLP技术提取关键词、主题
- 评分算法：基于IELTS/TOEFL写作评分标准

---

### 2.4 英语翻译模块

#### 功能描述
提供高质量的英汉互译功能，并辅助翻译学习。

#### 核心功能
1. **即时翻译**
   - 支持文本输入翻译
   - 支持语音翻译
   - 支持拍照翻译（OCR识别）
   - 支持对话翻译（实时语音互译）

2. **翻译学习**
   - 提供多种翻译结果对比
   - 解释翻译技巧和难点
   - 收藏经典翻译案例

3. **专业领域翻译**
   - 商务、法律、医学、技术等领域术语库
   - 支持用户上传专业文档翻译

4. **翻译记忆**
   - 保存翻译历史
   - 建立个人翻译记忆库
   - 相似句子自动匹配

#### 技术实现
- 翻译引擎：Google Translate API / DeepL API / 百度翻译API
- OCR识别：Tesseract / Google Vision API
- 术语库：Elasticsearch全文检索

---

### 2.5 论坛交流模块

#### 功能描述
打造英语学习社区，促进用户互动和交流。

#### 核心功能
1. **板块分类**
   - 学习经验分享
   - 资源分享（资料、工具、课程推荐）
   - 问答求助
   - 打卡圈子（每日学习打卡）
   - 英语角（自由讨论）

2. **互动功能**
   - 发帖、回复、点赞、收藏
   - @提醒功能
   - 私信功能
   - 关注感兴趣的用户

3. **内容管理**
   - 帖子搜索
   - 热门推荐
   - 精华帖标记
   - 举报和审核机制

4. **积分和等级系统**
   - 发帖、回帖获得积分
   - 等级晋升解锁更多功能
   - 排行榜展示

#### 技术实现
- 后端：Node.js + Express + MongoDB（适合论坛类非结构化数据）
- 实时通信：Socket.io（私信、通知）
- 搜索：Elasticsearch（全文搜索）

### 3.2 技术栈选型

#### 前端
- **移动端**：React Native（跨平台，开发效率高）
- **Web端**：Vue 3 + Element Plus（管理后台）
- **状态管理**：Redux（React Native）/ Pinia（Vue）
- **网络请求**：Axios + 请求拦截器

#### 后端
- **主框架**：Spring Boot 3.x（Java生态成熟）
- **微服务选项**：Spring Cloud Alibaba（后期可扩展）
- **ORM**：MyBatis-Plus（灵活、易上手）
- **API文档**：SpringDoc OpenAPI（Swagger替代）

#### 数据库
- **关系型**：MySQL 8.0（用户信息、单词数据、翻译记录）
- **文档型**：MongoDB 6.0（论坛帖子、聊天记录）
- **缓存**：Redis 7.0（会话、热点数据缓存）
- **搜索**：Elasticsearch 8.x（论坛搜索、单词搜索）

#### AI服务
- **大模型接入**：OpenAI API / 文心一言API / 通义千问API
- **语音识别**：Azure Speech Services
- **语音合成**：Azure TTS
- **翻译引擎**：DeepL API（质量高）/ 谷歌翻译API（免费额度）

#### 基础设施
- **版本控制**：Git + GitHub
- **容器化**：Docker + Docker Compose
- **CI/CD**：GitHub Actions / Jenkins
- **监控**：Spring Boot Actuator + Prometheus + Grafana
- **日志**：ELK Stack（Elasticsearch + Logstash + Kibana）

## 四、数据库设计

### 4.1 核心表结构

#### 用户相关
```sql
-- 用户表
CREATE TABLE `user` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `username` VARCHAR(50) NOT NULL UNIQUE,

  `password_hash` VARCHAR(255) NOT NULL,
  `avatar_url` VARCHAR(500),
  `level` INT DEFAULT 1,
  `points` INT DEFAULT 0,
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME ON UPDATE CURRENT_TIMESTAMP
);

-- 用户学习档案
CREATE TABLE `user_profile` (
  `user_id` BIGINT PRIMARY KEY,
  `total_words_learned` INT DEFAULT 0,
  `vocabulary_size` INT DEFAULT 0,
  `writing_count` INT DEFAULT 0,
  `chat_count` INT DEFAULT 0,
  `current_streak` INT DEFAULT 0,
  `longest_streak` INT DEFAULT 0,
  FOREIGN KEY (`user_id`) REFERENCES `user`(`id`)
);
```

#### 单词学习相关
```sql
-- 单词表
CREATE TABLE `word` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `word` VARCHAR(100) NOT NULL,
  `phonetic_uk` VARCHAR(100),
  `phonetic_us` VARCHAR(100),
  `definition_cn` TEXT,
  `definition_en` TEXT,
  `example_sentence` TEXT,
  `difficulty_level` INT,
  `wordbook_id` BIGINT,
  INDEX `idx_word` (`word`),
  INDEX `idx_wordbook` (`wordbook_id`)
);

-- 用户单词学习记录
CREATE TABLE `user_word_record` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `word_id` BIGINT NOT NULL,
  `mastery_level` INT DEFAULT 0,
  `review_count` INT DEFAULT 0,
  `next_review_time` DATETIME,
  `last_review_time` DATETIME,
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY `uk_user_word` (`user_id`, `word_id`),
  FOREIGN KEY (`user_id`) REFERENCES `user`(`id`),
  FOREIGN KEY (`word_id`) REFERENCES `word`(`id`)
);
```

## 五、API接口设计

### 5.1 用户模块
```
POST   /api/v1/users/register          注册
POST   /api/v1/users/login             登录
GET    /api/v1/users/profile           获取用户信息
PUT    /api/v1/users/profile           更新用户信息
POST   /api/v1/users/avatar           上传头像
```

### 5.2 单词学习模块
```
GET    /api/v1/wordbooks               获取词库列表
GET    /api/v1/wordbooks/{id}/words   获取词库单词列表
POST   /api/v1/words/learn            开始学习
POST   /api/v1/words/review           提交复习结果
GET    /api/v1/words/statistics       学习统计
```

### 5.3 AI对话模块
```
POST   /api/v1/chat/start             开始对话
POST   /api/v1/chat/message           发送消息
GET    /api/v1/chat/history           获取对话历史
DELETE /api/v1/chat/{id}             删除对话
```

### 5.4 写作模块
```
POST   /api/v1/writing/submit         提交作文
GET    /api/v1/writing/correction     获取批改结果
GET    /api/v1/writing/history        写作历史
GET    /api/v1/writing/models         获取范文
```

### 5.5 翻译模块
```
POST   /api/v1/translate/text         文本翻译
POST   /api/v1/translate/voice       语音翻译
POST   /api/v1/translate/image       图片翻译
GET    /api/v1/translate/history     翻译历史
```

### 5.6 论坛模块
```
GET    /api/v1/forum/posts           获取帖子列表
GET    /api/v1/forum/posts/{id}      获取帖子详情
POST   /api/v1/forum/posts           发帖
PUT    /api/v1/forum/posts/{id}      更新帖子
DELETE /api/v1/forum/posts/{id}      删除帖子
POST   /api/v1/forum/posts/{id}/reply  回复帖子
```

### 6.2 主要页面

3. **学习页**：单词卡片、对话界面、写作界面等
4. **社区页**：论坛帖子列表 + 发帖入口
5. **个人中心**：学习档案 + 设置

### 6.3 交互设计

## 九、风险与挑战

### 9.1 技术风险
- **AI API成本**：大模型调用费用可能较高
  - 对策：缓存常见对话、使用更便宜的模型
- **语音识别准确率**：口音识别可能存在问题
  - 对策：支持多种语音识别引擎切换

### 9.2 竞争风险
- 市场已有多款英语学习App（如扇贝、百词斩）
- 对策：差异化竞争，强化AI互动和社区氛围

### 9.3 合规风险
- 用户数据隐私保护
- AI生成内容审核
- 对策：遵守《个人信息保护法》，建立内容审核机制
